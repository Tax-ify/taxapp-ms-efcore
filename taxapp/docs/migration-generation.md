# Code First migration generation

How EF Core scaffolds migration files when you run `dotnet ef migrations add` or PMC `Add-Migration`. Focus: **model snapshots**, **`.Designer.cs` files**, and how they relate to the main migration class. New to EF Core migrations? Start with [migration-study-guide.md](migration-study-guide.md) for a conceptual overview first.

## Summary

`dotnet ef migrations add` produces **three files**:

| File | Purpose |
|------|---------|
| `{MigrationId}.cs` | `Up` / `Down` methods (schema changes as `MigrationOperation`s) |
| `{MigrationId}.Designer.cs` | Partial migration class with `[Migration]` metadata and a frozen **target model** |
| `{Context}ModelSnapshot.cs` | Single `ModelSnapshot` subclass; baseline for the **next** diff |

**Designer** and **snapshot** do not use separate model-to-code pipelines. Both call **`CSharpSnapshotGenerator`** to emit fluent `ModelBuilder` configuration from the current `IModel`. The snapshot is **rewritten on every new migration**; each migration’s Designer keeps that migration’s target model for history.

The main migration `.cs` file is generated separately via **`CSharpMigrationOperationGenerator`** from differences computed by **`MigrationsModelDiffer`**.

## Call flow

```
dotnet ef migrations add
  → MigrationsAddCommand (src/ef/Commands/)
  → OperationExecutor.AddMigration (src/EFCore.Design/Design/)
  → MigrationsOperations.AddMigration (src/EFCore.Design/Design/Internal/)
  → MigrationsScaffolder.ScaffoldMigration
  → MigrationsScaffolder.Save  → writes 3 files to disk
```

PMC `Add-Migration` uses the same `OperationExecutor` path.

## Orchestration: `MigrationsScaffolder`

**`ScaffoldMigration`** (`MigrationsScaffolder.cs`):

1. Load previous model from existing snapshot: `SnapshotModelProcessor.Process(modelSnapshot?.Model)`.
2. Diff against current context model: `MigrationsModelDiffer.GetDifferences` → `upOperations` / `downOperations`.
3. Select code generator: `MigrationsCodeGeneratorSelector.Select(language)` → typically `CSharpMigrationsGenerator`.
4. Generate three code strings:
   - `GenerateMigration` → main `.cs`
   - `GenerateMetadata` → `.Designer.cs`
   - `GenerateSnapshot` → `*ModelSnapshot.cs`
5. Return `ScaffoldedMigration` with all strings.

**`Save`** writes:

- `{MigrationId}.cs` ← `MigrationCode`
- `{MigrationId}.Designer.cs` ← `MetadataCode` (filename built as `MigrationId + ".Designer" + FileExtension`)
- `{SnapshotName}.cs` ← `SnapshotCode`

Snapshot name defaults to `{ContextName}ModelSnapshot` (e.g. `BlogContext` → `BlogContextModelSnapshot`). An existing snapshot class name is reused when present.

## `.Designer.cs` generation

**Method:** `CSharpMigrationsGenerator.GenerateMetadata`

Produces a **partial class** of the migration with attributes and embedded model:

```csharp
// <auto-generated />
[DbContext(typeof(MyContext))]
[Migration("20240517120000_MyMigration")]
partial class MyMigration
{
    protected override void BuildTargetModel(ModelBuilder modelBuilder)
    {
#pragma warning disable 612, 618
        // ... CSharpSnapshotGenerator output ...
#pragma warning restore 612, 618
    }
}
```

Details:

- `// <auto-generated />` header.
- `#pragma warning disable/restore 612, 618` around generated model code.
- Body: `CSharpSnapshotGenerator.Generate("modelBuilder", targetModel, builder)`.

At runtime, `Migration.TargetModel` is built from this Designer metadata.

## Model snapshot generation

**Method:** `CSharpMigrationsGenerator.GenerateSnapshot`

Produces a class inheriting `ModelSnapshot`:

```csharp
// <auto-generated />
[DbContext(typeof(MyContext))]
partial class MyContextModelSnapshot : ModelSnapshot
{
    public override string LastMigrationId => "20240517120000_MyMigration";

    protected override void BuildModel(ModelBuilder modelBuilder)
    {
#pragma warning disable 612, 618
        // ... CSharpSnapshotGenerator output ...
#pragma warning restore 612, 618
    }
}
```

- `LastMigrationId` helps detect merge conflicts between migration branches.
- Body again uses `CSharpSnapshotGenerator.Generate`.

**Runtime:** `MigrationsAssembly` finds the single `ModelSnapshot` subclass in the migrations assembly. `ModelSnapshot.Model` lazily calls `BuildModel` via `ModelBuilder`.

**Design-time note:** Snapshots use a property-bag style model (`Dictionary<string, object>` in generated code), not necessarily the app’s CLR entity types. `SnapshotModelProcessor.Process()` fixes up older snapshots for backward compatibility.

## Shared core: `CSharpSnapshotGenerator`

Single place that turns `IModel` into fluent `ModelBuilder` C# (~2000 lines).

`Generate(modelBuilderName, model, stringBuilder)`:

- Model-level annotations (including `ProductVersion`).
- Sequences.
- Entity types in hierarchical order (properties, keys, indexes, relationships, etc.).
- Uses `AnnotationCodeGenerator` for annotation serialization.

Both Designer (`BuildTargetModel`) and Snapshot (`BuildModel`) are thin wrappers around this generator.

---

## Main migration `.cs` (context)

**Method:** `CSharpMigrationsGenerator.GenerateMigration`

Emits `partial class {MigrationId} : Migration` with `Up` / `Down` calling `CSharpMigrationOperationGenerator.Generate` for each `MigrationOperation` (`CreateTable`, `AddColumn`, etc.) from `MigrationsModelDiffer`.

---

## Runtime compilation (optional path)

`CSharpMigrationCompiler` (`src/EFCore.Design/Migrations/Design/CSharpMigrationCompiler.cs`) parses all three generated sources together via Roslyn when migrations are compiled in-memory (runtime migrations scenario). Normal `migrations add` only writes files to disk.

## Pipeline diagram

```
                    MigrationsScaffolder.ScaffoldMigration
                                    |
          +-------------------------+-------------------------+
          |                         |                         |
   MigrationsModelDiffer    CSharpMigrationsGenerator   (current IModel)
          |                         |
          |              +----------+----------+----------+
          |              |          |          |
          |         GenerateMigration  GenerateMetadata  GenerateSnapshot
          |              |          |          |
          |              v          v          v
          |         Migration.cs  .Designer.cs  ModelSnapshot.cs
          |              |          |          |
          +--------------+          +----------+
     MigrationOperations              |
          |                    CSharpSnapshotGenerator
          v                            |
 CSharpMigrationOperationGenerator    +----> (same model C# in both)
```
